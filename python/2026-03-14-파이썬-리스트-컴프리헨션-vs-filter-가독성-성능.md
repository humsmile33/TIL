### Context
* 문제: 리스트 컴프리헨션(list comprehension) 내부에 조건문(if)을 여러 개 넣으면 가독성이 떨어짐. 예: [x for x in lst if a if b]
* 고민: 가독성 때문에 filter() 사용을 고려했으나, 대량 데이터 처리 시 성능 차이가 난다는 관찰이 있음. (키워드: list comprehension, filter, lambda, generator expression, CPython)
* 목표: 가독성(비전공자 우선)과 성능(대량 데이터) 중 상황에 맞는 권장 패턴과 벤치마크 방법 제공.

### Core
* 예제 데이터와 조건 함수
```python
import random
from time import perf_counter

N = 3_000_000
data = [random.randint(0, 10_000) for _ in range(N)]

def cond_a(x):
    return x % 2 == 0

def cond_b(x):
    return x % 3 == 0
```
* 방법 A — 리스트 컴프리헨션 (가독성·보통 빠름)
```python
start = perf_counter()
res_comp = [x for x in data if cond_a(x) if cond_b(x)]
elapsed = perf_counter() - start
print('comprehension:', elapsed, 'len=', len(res_comp))
```
* 방법 B — filter + lambda (가독성 낮음, 호출 오버헤드 있음)
```python
start = perf_counter()
res_filter_lambda = list(filter(lambda x: cond_a(x) and cond_b(x), data))
elapsed = perf_counter() - start
print('filter+lambda:', elapsed, 'len=', len(res_filter_lambda))
```
* 방법 C — filter + named function (filter가 내장 C 루프를 활용하면 빠를 때가 있음)
```python
def both(x):
    return cond_a(x) and cond_b(x)

start = perf_counter()
res_filter_fn = list(filter(both, data))
elapsed = perf_counter() - start
print('filter+fn:', elapsed, 'len=', len(res_filter_fn))
```
* 방법 D — 제너레이터/지연평가 (메모리 효율)
```python
start = perf_counter()
res_gen = (x for x in data if cond_a(x) and cond_b(x))
# 필요 시 바로 소비하지 않으면 메모리 절약
count = sum(1 for _ in res_gen)
elapsed = perf_counter() - start
print('generator (lazy):', elapsed, 'count=', count)
```
* 실행 및 재현 가능한 벤치마크 팁
```python
# timeit 모듈 사용 권장(환경에 따라 편차 있음)
import timeit
setup = '\n'.join([
    'import random',
    'from __main__ import data, cond_a, cond_b, both'
])
stmt_comp = '[x for x in data if cond_a(x) if cond_b(x)]'
stmt_filter = 'list(filter(lambda x: cond_a(x) and cond_b(x), data))'

print('comp timeit:', timeit.timeit(stmt_comp, setup=setup, number=3))
print('filter timeit:', timeit.timeit(stmt_filter, setup=setup, number=3))
```
* 추가 성능/설계 팁
* 명확한 조건은 하나의 if문에 and로 묶어 표현하는 것이 가독성에 도움이 됨: [x for x in data if cond_a(x) and cond_b(x)]
* filter는 Python 3에서 iterator를 반환하므로(메모리 절약) 파이프라인 처리(streaming)에 유리함. (filter returns an iterator)
* filter에 넘기는 함수가 C로 구현된 빌트인(built-in)일 경우(예: str.isdigit 등) 호출 오버헤드가 작아 유리할 수 있음
* 수치·벡터 연산에는 순수 Python 반복이 아닌 NumPy/Pandas 벡터화 사용 권장(대규모 데이터에서 압도적 우위)

### Insight
* 성능 관찰
* 리스트 컴프리헨션은 CPython에서 C-레벨의 리스트 생성/append 최적화를 이용하므로, 동일한 조건을 평가할 때 filter+lambda보다 일반적으로 빠른 경향이 있음.
* 그러나 filter는 iterator(지연평가)를 반환하므로 메모리 제약이 있는 파이프라인 처리에서는 유리하다.
* filter에 전달되는 함수가 순수 Python 함수(lambda 포함)라면 호출 오버헤드가 발생해 느려질 수 있지만, C-구현(built-in) 함수를 사용하면 성능 이점이 있을 수 있음.

* 가독성 권장
* 비전공자(또는 초심자)에게는 가독성을 우선 권장: 조건을 분해해서 이름 있는 함수로 추출하거나(예: is_even_and_div3), 한 줄로 작성하더라도 and로 묶은 형태가 이해하기 쉬움.
* 예: def is_valid(x): return cond_a(x) and cond_b(x)
  그리고 [x for x in data if is_valid(x)] 처럼 쓰면 의도 전달이 명확함.

* 성능이 중요할 때
* 핫스팟(프로파일러로 찾은 부분)에 한해 리스트 컴프리헨션이나 C-레벨 구현(NumPy, builtins)을 사용해 최적화하세요. 전체 코드에서 무분별히 바꾸는 것은 권장되지 않습니다.
* 메모리 한계가 명확하면 generator expression 혹은 filter(iterator)를 사용해 메모리 소비를 줄이세요.

* 요약 권장 패턴
* 가독성 우선(비전공자): 리스트 컴프리헨션 + 명명된 조건 함수
* 성능(대량 데이터) 우선: 리스트 컴프리헨션(대부분 빠름) 또는 벡터화(NumPy/Pandas). 메모리 제약 시 generator/filter
